Portfolio 6 - Boring + Not-Lottery Screen with Inverse-Vol Weights and Crash Hedge
========================================================================================

GOAL
- Pick ~50 stocks from S&P 500 + Nasdaq-100 by combining low realized vol, low max
  one-day return (skip lottery names), and (if fundamentals CSV present)
  high gross_profit / total_assets.
- Weight survivors by inverse realized volatility, cap each stock at MAX_WEIGHT_PER_STOCK.
- Scale the stock sleeve so realized portfolio vol roughly matches VOL_TARGET_ANNUAL,
  capped at MAX_LEVERAGE.
- Add a small gold sleeve (GLD by default) and an optional trend-following hedge
  (e.g. DBMF / KMLM) - left empty in config until that ticker is backfilled.
- Rebalance once per month, on the first tick of each calendar month.

PIPELINE
1. _load_universe reads src/portfolios/portfolio_6/universe.json. That file is
   generated by scripts/APIs/build_portfolio6_universe.py from the FMP S&P 500
   and Nasdaq-100 endpoints. Hedge tickers are merged into TICKERS at __init__.
2. _collect_returns pulls VOL_LOOKBACK_DAYS + 30d of daily closes per candidate
   via context.Market[ticker].History and returns daily-return series.
3. score_universe (screener.py) builds a composite rank-sum:
      vol_rank (asc) + max_one_day_rank (asc)
        + optional gross_profit/total_assets rank (desc) from fundamentals CSV.
4. inverse_vol_weights produces 1/vol weights with iterative capping at
   MAX_WEIGHT_PER_STOCK.
5. deflated_sharpe_ratio logs the probability the selected basket's SR is real
   given multiple-testing inflation (Lopez de Prado 2014). DSR_MIN_PROB controls
   when we log a "may be noise" warning.
6. vol_target_scale scales the stock sleeve so realized sleeve vol ~= VOL_TARGET_ANNUAL,
   then GLD_WEIGHT and TREND_HEDGE_WEIGHT are added. Total is capped at MAX_LEVERAGE.
7. _execute_orders walks the held positions and the target dict; for every name
   whose weight drift exceeds REBALANCE_DRIFT_THRESHOLD it calls the executor
   directly with the TARGET weight (not the current weight, which is what the
   default context.buy/sell helpers send).

KNOWN LIMITATIONS (v1)
- Universe is a current snapshot of S&P 500 and Nasdaq-100; there is no historical
  constituents data, so backtests over many years suffer survivor bias.
- Fundamentals are sourced via scripts/APIs/fmp_fundamentals.py to a local CSV
  in <repo>/fundamentals/fundamentals.csv. No DB schema change. The CSV reflects
  the latest available statements, not point-in-time fundamentals.
- Trend-following hedge ticker is config-driven; default empty. DBMF / KMLM are
  not currently backfilled into market_data, so set TREND_HEDGE_TICKER only after
  backfilling the chosen ticker.
- Event mode only. No fast-mode vector adapter for v1.

FILES IN THIS PACKAGE
- strategy.py     Portfolio6Strategy (extends BasePortfolio).
- screener.py     scoring, inverse-vol weights, vol-target scaler, DSR.
- config.json     PORTFOLIO_ID=6, hedge weights, vol target, leverage cap, etc.
- universe.json   Generated by scripts/APIs/build_portfolio6_universe.py.

EXTERNAL SCRIPTS
- scripts/APIs/build_portfolio6_universe.py   Pulls S&P 500 + Nasdaq-100 from FMP.
- scripts/APIs/fmp_fundamentals.py            Pulls ROE / gross_profit / total_assets
                                              and writes fundamentals/fundamentals.csv.
